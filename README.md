# Blinkray

Веб-панель на aiohttp для управления VLESS-сервером на базе xray-core:
включение/выключение сервиса, настройка (адрес, порт, режим инбаунда),
управление клиентами с генерацией `vless://` ссылок для v2rayNG, раздача
apk-файла v2rayNG. Слушает HTTPS на порту `10077` с самоподписным
сертификатом, вход по паролю.

Поддерживаются два режима инбаунда, переключаются флагом в веб-интерфейсе
(раздел «Настройки подключения»):

- **VLESS + REALITY** (TCP) — без собственного TLS-сертификата, маскируется
  под настоящий сайт (`dest`/SNI), сложнее для DPI;
- **VLESS + WS + TLS** — трафик поверх WebSocket с самоподписным
  сертификатом и `allowInsecure`, полезно там, где REALITY по какой-то
  причине не заходит.

Переключение режима не трогает список клиентов — у каждого просто меняется
формат `vless://`-ссылки под текущий режим.

Панель ставится **на сам VPS**, рядом с xray-core — она сама пишет `config.json`
для xray и управляет им через systemd.

## Установка (один скрипт)

На чистом VPS (Ubuntu/Debian, есть также best-effort поддержка dnf/yum):

```bash
git clone https://github.com/lbestd/blinkray.git
cd blinkray
./install.sh
```

Скрипт сам, без дополнительных вопросов:

- если запущен не из `/opt/blinkray` — перенесёт себя туда (`cp -a`, включая
  `.git`) и перезапустится оттуда. Это принципиально: `systemd` делает `chdir`
  в `WorkingDirectory` уже от имени непривилегированного пользователя
  `blinkray`, а не root, поэтому клон в `/root/blinkray` (права `700` на
  `/root`) не запустился бы, сколько ни меняй владельца самой папки —
  `/opt` устраняет эту проблему раз и навсегда;
- поставит `python3`, `venv`, `openssl`, `curl`, `unzip`;
- создаст системных пользователей `blinkray` (панель) и `xray` (сам xray-core),
  добавит `xray` в группу `blinkray`, чтобы он мог читать сгенерированные
  панелью `config.json` и TLS-ключ;
- скачает актуальный `xray-core` с GitHub под архитектуру сервера (или возьмёт
  бинарь из `vendor/xray`, если вы положили его туда заранее — например свой
  `./xray` с этой машины);
- создаст venv и поставит зависимости;
- сгенерирует случайный пароль в `/etc/blinkray.env` (если не задать свой —
  см. ниже) и распечатает его в конце;
- настроит `sudo`-правило (панель без пароля может только start/stop/restart/
  enable/disable `xray.service` — ничего больше);
- создаст и запустит systemd-юниты `blinkray.service` и `xray.service`.

Скрипт идемпотентен — его можно перезапускать, существующий пароль он не
тронет.

Свой пароль вместо случайного:

```bash
PANEL_PASSWORD='мой-пароль' ./install.sh
```

После установки:

1. Откройте `https://<IP-сервера>:10077` (браузер предупредит про
   самоподписной сертификат — это ожидаемо, жмите «продолжить»).
2. Войдите паролем, который скрипт распечатал (или своим, если задавали).
3. В разделе **«Настройки подключения»** укажите публичный адрес сервера,
   порт, WS-путь и SNI, сохраните.
4. В разделе **«Сервер»** нажмите **«Запустить»**.
5. Добавьте клиентов в разделе **«Клиенты»** — для каждого будет готова
   `vless://` ссылка, которую можно вставить прямо в v2rayNG.
6. При желании загрузите сам apk v2rayNG в разделе **«Пакет v2rayNG»** —
   появится ссылка на скачивание.

## Обновление

```bash
cd /opt/blinkray   # install.sh уже перенёс код сюда при первом запуске
git pull
./install.sh          # безопасно перезапускать — не трогает пароль/данные
systemctl restart blinkray
```

## Что где лежит после установки

- Код и venv — всегда `/opt/blinkray`, независимо от того, куда изначально
  делали `git clone` (см. выше).
- Состояние панели (настройки, список клиентов, сертификаты, apk) — в
  `/opt/blinkray/data/`, не в git (см. `.gitignore`).
- Пароль и прочие переменные окружения — `/etc/blinkray.env`.
- sudoers-правило — `/etc/sudoers.d/blinkray`.
- systemd-юниты — `/etc/systemd/system/blinkray.service` и `xray.service`.

## Firewall

Откройте на сервере:
- `10077/tcp` — веб-панель (по-хорошему ограничить по IP, если провайдер
  позволяет security group/firewall правила);
- порт VLESS-инбаунда, заданный в настройках (по умолчанию `443/tcp`).

## Переменные окружения

Заданы в `/etc/blinkray.env` (создаётся `install.sh`). Полный список — в
`.env.example`. Единственная обязательная — `PANEL_PASSWORD`.

## Локальный запуск для разработки (без systemd/xray)

```bash
python3 -m venv venv && ./venv/bin/pip install -r requirements.txt
PANEL_PASSWORD=test PANEL_DATA_DIR=$(pwd)/data ./venv/bin/python3 app.py
```

Панель поднимется на `https://0.0.0.0:10077` с самоподписным сертификатом.
Без установленного xray и systemd-юнита кнопки управления сервером будут
возвращать ошибку — это нормально при локальном тестировании самой панели.

## Ручная установка / кастомизация

Если не хочется единого скрипта (свои имена пользователей, другой
`WorkingDirectory`, отдельный хост под панель и т.д.) — откройте `install.sh`,
это обычный bash-скрипт из последовательных шагов (пользователи → xray-бинарь
→ venv → права → `/etc/blinkray.env` → sudoers → systemd-юниты → `systemctl
enable --now`). Возьмите из него нужные куски и адаптируйте под себя.
